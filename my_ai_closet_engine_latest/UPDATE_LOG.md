### [2026-05-20] 추천 상세 조건(원하는 분위기, 격식 정도, 활동량) DB 키워드 기반 지능형 매핑 고도화
- **목적:** 사용자가 사이드바 상세 조건(원하는 분위기, 격식 정도, 활동량) 필터를 조작했을 때, 실제 데이터베이스(`reference_styles.json`)에 존재하지 않는 텍스트로 인해 매칭 가산점이 반영되지 않거나 어긋나는 현상과 미충족 진단 버그를 해결함.
- **주요 변경 사항:**
  - **상세 무드 필터 매핑 연동 수정 (`core/recommender.py`):**
    - `tpo_mood` 및 `desired_mood`의 "포멀/클래식", "스트릿" 등과 같은 옵션을 실제 DB 무드 분포 데이터인 `"놈코어(normcore)"`, `"캐주얼(casual)"`, `"스포티(sporty)"`, `"로맨틱(romantic)"`, `"레트로(retro)"` 등으로 지능형 변환 매핑하여 정합성 보장.
    - 격식 정도(`formality_level`)를 분석하여 높음(정장/클래식) 설정 시 놈코어 무드와 격식 있는 아우터(자켓, 코트 등) 조합을 선별하고, 낮음 설정 시 스포티/애슬레저 위주로 선별되도록 가중치 룰을 현대화.
    - 활동량(`activity_level`)과 레퍼런스 스타일 간의 연동 룰 역시 실제 데이터 라벨 분포에 맞춰 개선.
  - **조건 완화 진단 및 처방 오판정 버그 수정:**
    - 원하는 분위기를 잘 매칭했음에도 불구하고 DB 단어 불일치로 인하여 항상 미충족 조건 경고창과 함께 보완 의상 쇼핑 처방이 노출되던 미충족 판정(`unmet_conditions`) 로직을 실제 개선된 놈코어/캐주얼 등의 키워드 검증 방식과 통일시켜 오판정을 완벽히 차단.

### [2026-05-20] 한겨울(-5도) 코디 추천 오류 해결 및 TPO 매칭 정밀도 개선 (핫픽스)
- **목적:** 영하 5도 등의 겨울 날씨에 얇은 봄/가을용 자켓이 추천되는 계절 부적합 오류를 수정하고, 구매 후보에 있는 코트 및 내 옷장의 패딩이 정상적으로 매칭되도록 추천 알고리즘을 개선함.
- **주요 변경 사항:**
  - **아이템 매칭 시 계절/기온 가중치 도입 (`core/recommender.py`):**
    - 날씨 기온 강도(`weather_data["warmth_level"]`)에 맞춰 아이템별 가중치 연산 추가.
    - 영하의 추운 겨울 날씨일 때 두꺼운 의류(`패딩`, `코트` 등) 또는 겨울/사계절 의상에 **+20점** 가산을 제공하고, 얇은 봄/가을 전용 의상에는 **-10점** 감점 처리를 하여 계절 매칭의 무결성을 보장.
    - 반대로 더운 여름 날씨에는 겨울 의류에 대해 **-30점** 감점을 부여하여 원천 배제.
  - **TPO 카테고리 매칭 키워드 최적화:**
    - 기존 DB 레퍼런스 무드 명칭에 "포멀", "오피스" 등의 단어가 누락되어 있어 출근  TPO 가점이 적용되지 않던 현상 해결.
    - 실제 데이터 무드 키워드인 `"놈코어"`, `"normcore"` (오피스 대용) 및 `"romantic"` (데이트 대용)을 매칭 기준에 추가 반영하여 추천 적합도 점수가 정상적으로 상승하도록 보완.

### [2026-05-20] 팀장 SQL 스키마 규격(category_code, style_tags) 대응 및 데이터 마이그레이션
- **목적:** RDBMS(관계형 데이터베이스) 마스터 테이블 규격인 `category_code` 및 `style_tags`를 지원하여, 기존 로컬 엔진 기능의 손실 없이 백엔드 서버 연동 시 데이터를 1:1로 손쉽게 마이그레이션할 수 있도록 구조를 확장함.
- **주요 변경 사항:**
  - **SQL 호환 필드 자동 추출 엔진 구축 (`core/analyzer.py`):**
    - 성별(M/F) 및 스타일(STREET, MINIMAL, OFFICE, CAMPUS)을 매핑하는 규칙을 통해 `category_code`를 자동 생성하는 `generate_sql_compatible_fields` 헬퍼 함수 구현.
    - 한국어 의류 상세명, 색상, 계절, 스타일명을 영문 단어들로 변환해 쉼표로 잇는 `style_tags` 자동 추출 로직 반영.
    - 로컬 스캔(`sync_user_closet_local`, `sync_shopping_candidates_local`) 및 데모 데이터 생성 시 자동으로 신규 필드를 생성하도록 보완.
  - **안전한 DB 마이그레이션 유틸리티 구현:**
    - 기존 데이터(JSON)에 신규 필드가 누락되어 있을 경우, 실행 즉시 자동으로 데이터를 보완해 주는 `migrate_existing_data_to_sql_format` 구현하여 앱 시작 시 구동.
  - **Streamlit 화면 에디터 확장 (`app.py`):**
    - `내 옷장` 및 `구매 후보` 데이터 테이블 에디터(`st.data_editor`)에 `category_code`와 `style_tags` 열을 새로 노출하여 수동 수정 및 변경 저장을 지원하도록 고도화.

### [2026-05-20] 원클릭 로컬 실행 파일 생성 및 윈도우 배치 스크립트 배포
- **목적:** 사용자가 터미널 명령을 직접 입력하지 않고도 더블 클릭만으로 로컬 가상환경에서 Streamlit 앱을 손쉽게 구동할 수 있도록 환경 자동 감지 및 패키지 체크 로직이 내장된 실행 링크(배치 파일)를 마련함.
- **주요 변경 사항:**
  - **원클릭 로컬 실행기 (`run.bat`) 신규 생성:**
    - 더블 클릭 한 번으로 `.venv` 가상환경을 알아서 탐색 및 활성화하는 윈도우 배치 스크립트 추가.
    - 실행 환경에 필수 모듈인 `streamlit` 등이 미설치되어 있는 경우 `requirements.txt`에 기록된 라이브러리들을 일괄 자동 설치한 뒤 안전하게 앱을 실행하도록 방어 코드 탑재.
    - 터미널 출력 화면에서 한글이 깨지지 않도록 UTF-8 코드 페이지(`chcp 65001`)를 적용하여 매끄러운 메시지 제공.

### [2026-05-19] 스타일명 현대화, UI 개선, 히스토리 NameError 수정 및 프론트엔드 연동 문서 구축
- **목적:** 구식 연대 명칭 제거와 최신 패션 트렌드에 걸맞은 스타일명 반영, 중복된 분위기 입력란 삭제, 히스토리 보기 시의 NameError 해결 및 프론트엔드 개발자가 쉽게 최상급 UI를 개발할 수 있도록 설계 가이드 마련.

  - **스타일명 현대화 및 데이터 마이그레이션:**
    - `core/analyzer.py` 내 연대 표기("2010년대", "2019년대") 결합 로직 제거.
    - 스타일 한글 카테고리명을 현대 감각(스포티 데일리, 모던 놈코어, 올드머니 클래식, 뉴트로 스트릿 등)으로 개편.

  - **마네킹 착장 보드 HTML 렌더링 수정:**
    - HTML 내 들여쓰기 공백으로 인해 마크다운 파서가 코드 블록으로 렌더링하던 현상을 수정하기 위해 반환값 행마다 `.strip()` 처리를 적용하여 정상 그래픽 렌더링 확보.
  - **추천 설정 UI 단순화 및 중복 위젯(오늘의 코디 분위기) 제거:**

  - **히스토리 복원 영역의 NameError 수정:**
    - 저장된 코디 히스토리를 불러올 때 `category_order` 변수가 스코프 외부로 판정되어 발생하던 `NameError` 오류를 해결하기 위해 `main()` 최상단에서 이를 Main 스코프로 정의.
  - **개발자용 연동 가이드 (`docs/ENGINE_UI_UX_GUIDE.md`):**

  - **구매 후보 의류 성별 일관성(남성복) 확보:**
    - 남성인 사용자의 옷장 구성과 성별 매치가 이루어지도록 임의로 생성되어 있던 구매 후보 의류(브라운 코트, 베이지 블레이저)를 남성용 의류 이미지로 새로 생성하여 대체.
    - `shopping_candidates.json` DB의 `gender` 컬럼을 `"Man"`으로 일괄 변환하여 남성용 믹스 매치 추천이 자연스럽게 연동되도록 조치.
