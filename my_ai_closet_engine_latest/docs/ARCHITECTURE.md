# 🏗️ My AI Closet — 시스템 아키텍처 및 상용화 가이드

이 문서는 엔진의 전체 아키텍처, 데이터 흐름, 추천 알고리즘 파이프라인, 그리고 상용 서비스로의 확장 설계를 기술합니다.

---

## 1. 시스템 아키텍처 개요

본 시스템은 **UI ↔ Engine ↔ Data** 3계층 구조로 설계되어 있습니다.

```
[UI Layer: Streamlit] (app.py)
        │
        ▼ (사용자 입력 & 렌더링 호출)
┌──────────────────────────────────────────────────────────┐
│ [Engine Layer: core/]                                    │
│                                                          │
│  1. recommender.py (스타일 매칭, 조건 스코어링, 폴백, 처방) │
│  2. analyzer.py    (로컬 이미지 분석, 카테고리/색상 추론)    │
│                                                          │
└──────────────────────────────────────────────────────────┘
        │
        ▼ (JSON 데이터 동기화)
[Data Layer: datasets/] + JSON DB 파일들
```

### 핵심 설계 원칙
| 원칙 | 설명 |
| :--- | :--- |
| **100% 오프라인** | 외부 API·클라우드 의존성 0%. 모든 연산은 로컬에서 수행 |
| **역할 분리** | analyzer = 분석/분류, recommender = 추천/판단. UI는 순수 뷰 레이어 |
| **설정 중앙화** | `config.py` 한 곳에서 모든 경로를 관리 |
| **데이터 독립성** | 내 옷장 / 구매 후보 / 스타일 교과서 DB가 물리적으로 분리 |

---

## 2. 데이터 흐름 파이프라인

```
[이미지 폴더] ─→ analyzer.py ─→ [JSON DB] ─→ recommender.py ─→ [추천 결과]
                     │                                │
                     ▼                                ▼
              색상 추출 (Pillow)              스코어링 (규칙 기반)
              카테고리 분류                   색상 궁합 매칭
              라벨 메타데이터 파싱            체형 보완 가중치
                                             구매 가치 판단
```

### 스타일 교과서 분류 우선순위
1. **AI Hub JSON 라벨** (`labels/*.json`) → Q4201~Q4216 체크박스 기반 정밀 분류
2. **selected_inventory.json** → Woman 데이터셋 전용, 스타일 + 아이템 목록 기반
3. **이미지 이분할 분석** → 상단/하단 색상 차이로 "상의+하의" vs "원피스" 판별
4. **파일명 키워드** → `shirt`, `blazer`, `jeans` 등 매칭

## 3. 추천 알고리즘 상세 (4단계 파이프라인)

### 1단계: 스타일 교과서 스코어링
성별 필터링 후 각 레퍼런스에 점수를 부여합니다.

| 조건 | 가중치 |
| :--- | :--- |
| TPO 무드 수동 지정 매칭 | +30점 |
| 원하는 분위기 매칭 | +15점 |
| 격식 정도 매칭 | +15점 |
| 활동량 수준 매칭 | +15점 |
| TPO 자동 키워드 매칭 | +7~10점 |
| 날씨/계절 매칭 | +3~5점 |
| 개별 키워드 보너스 | +3점 |

### 2단계: 의상 풀(Pool) 동적 구성
추천 모드에 따라 매칭 대상을 다르게 조합합니다.
- **내 옷장만**: `my_closet.json`만 사용
- **구매 후보만**: `shopping_candidates.json`만 사용
- **믹스 매칭**: 두 DB 합산, 소장 의상에 우선순위 가산점(+4점)

### 3단계: 아이템 매칭 (색상 궁합 + 체형 보완)
| 조건 | 가중치 |
| :--- | :--- |
| 색상 궁합표 매칭 | +10점 |
| 선호 색상 일치 | +12점 |
| 소장 의상 우선 (믹스 모드) | +4점 |
| 체형 보완 (어두운 하의 등) | +3~5점 |

**기피 대상 필터링:**
- 색상/아우터/원피스/스커트 제외 옵션 적용
- 필터 후 후보가 0벌이면 자동 폴백(기피 조건 해제)

### 4단계: 구매 추천 판단 (Purchase Advice)
구매 후보 의류에 대해 내 옷장과 비교 분석합니다.
- **구매 추천 (높음) 🔥**: 호환 의상 ≥ 3개 & 중복 없음
- **구매 추천 (보통)**: 호환 의상 < 3개 & 중복 없음
- **보류 (중복 우려)**: 동일 카테고리+색상 의상이 이미 존재

## 4. 미충족 조건 진단 및 처방

추천 후 원하는 조건이 충족되지 못한 경우 자동 진단합니다.

| 진단 유형 | 처방 내용 |
| :--- | :--- |
| 원하는 분위기 미충족 | 해당 무드에 맞는 기본 아이템 추천 |
| 선호 색상 미충족 | 해당 색상의 베이직 의류 추천 |
| 기피 조건 강제 해제 | 뉴트럴 색상 대체 아이템 추천 |

## 6. 기능 추가/삭제 가이드

### 새로운 추천 조건 추가하기 (3단계)
1. **UI (`app.py`)**: 상세 조건 Expander에 `selectbox` 위젯 추가
2. **함수 파라미터 (`recommender.py`)**: `recommend_outfit`에 새 인자 추가 (기본값 권장)
3. **스코어링 룰**: 새 조건에 대한 가중치 계산 로직 추가

### 특정 조건 제거하기
1. UI 위젯 제거 → 2. 파라미터 제거/방치 → 3. 스코어링 로직 주석 처리

### 점수 밸런싱 조정
`recommender.py` 내 점수 가산 상수들만 수정하면 됩니다.

## 7. 상용화 확장 설계

### ① DB 마이그레이션 (JSON → RDB)
```sql
CREATE TABLE user_closet (
    item_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,
    category VARCHAR(20) NOT NULL,
    detail VARCHAR(100),
    color VARCHAR(20),
    season VARCHAR(50),
    style VARCHAR(30),
    image_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### ② 이미지 업로드 파이프라인
```
[모바일 앱] → [백엔드 서버] → [클라우드 스토리지(S3)] → [분석 엔진] → [DB 기록]
```

### ③ AI 고도화 경로
- **현재**: 규칙 기반 키워드 매칭 + 색상 궁합표
- **1단계**: 임베딩 벡터 기반 유사도 검색 (CLIP, FashionCLIP)
- **2단계**: LLM 연동 (Gemini/GPT)으로 자연어 스타일링 텍스트 생성
- **3단계**: 사용자 착용 이력 기반 개인화 추천 (협업 필터링)

### ④ 비동기 처리
대량 이미지 스캔 시 Streamlit UI 블로킹을 방지하기 위해 Celery 등의 비동기 작업 큐 도입을 권장합니다.
