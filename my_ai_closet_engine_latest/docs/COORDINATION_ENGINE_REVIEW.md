이 문서는 AI 코디네이션 엔진의 구조와 완성도를 최종 점검하고, 향후 다른 개발자가 엔진을 확장(기능 추가/삭제)하거나 서비스에 연동할 때 참고할 수 있도록 작성한 **유지보수 및 아키텍처 가이드**입니다.

## 1. 엔진 완성도 점검 (Completeness Check)

추천 엔진 및 분석 모듈을 정밀 분석한 결과, 기획된 스펙들이 완성도 있게 구현되어 있음을 확인하였습니다.

| 분석 영역 | 구현 상태 | 검증 내용 |
| :--- | :--- | :--- |
| **1. 5대 스타일 조건 필터링** | **완료** | `desired_mood`, `formality_level`, `preferred_color`, `avoid_options`, `activity_level` 입력 연동 및 **DB 라벨 호환 지능형 하이브리드 매핑** 반영 |
| **2. 안전 폴백 (Safety Fallback)** | **완료** | 기피 조건(아우터 제외, 원피스 제외, 특정 색상 피하기 등)으로 인해 조합 후보가 0벌이 될 경우, 자동으로 조건을 완화하여 추천이 중단되지 않도록 방지하는 안전망 구축 |
| **3. 미충족 조건 진단 및 제안** | **완료** | 옷장에 옷이 부족하여 원하는 분위기/색상이 미반영되었을 때, `unmet_conditions`로 감지하고 부족한 옷을 구매하도록 추천하는 `complementary_suggestions` 처방 |
| **4. 필수 기본템 진단 스캐너** | **완료** | 현재 옷장의 누적 수량을 스캔하여 꼭 필요한 기본 의류(아우터/상의/하의/뉴트럴 색상) 구비 상태 진단 및 추천 |
| **5. 구매 의사결정 도우미** | **완료** | 구매 후보 의류에 대해 '내 옷장의 호환 가능 색상 옷 개수'와 '중복 옷 존재 여부'를 연산하여 구매 가치 등급화 (`구매 추천(높음/보통)/보류`) |
| **6. 히스토리 로깅** | **완료** | 매칭된 코디 데이터(날짜, 날씨, 옷 정보, 실루엣 성별 등)를 `outfit_history.json`에 영속화하고 최신순으로 역정렬하여 복원하는 기능 완비 |

## 2. 엔진의 모듈별 역할 및 설계 구조 (Clean Architecture)

다른 개발자가 코드를 한눈에 파악하고 수정할 수 있도록 핵심 로직을 물리적으로 완벽히 격리하였습니다.

```
[UI Layer: Streamlit] (app.py)
        │
        ▼ (사용자 입력 & 렌더링 호출)
┌──────────────────────────────────────────────────────────┐
│ [Engine Layer: core/]                                    │
│                                                          │
│  1. recommender.py (스타일 매칭, 조건 스코어링, 폴백, 처방) │
│  2. analyzer.py    (로컬 이미지 분석, 카테고리/색상 추론)    │
│                                                          │
└──────────────────────────────────────────────────────────┘
        │
        ▼ (JSON 데이터 동기화)
[Data Layer: datasets/] (reference_styles.json, my_closet.json, shopping_candidates.json)
```

### 1) `core/recommender.py` (핵심 추천 엔진)
- **역할:** 날씨(기온) 분석, 레퍼런스 스타일 매칭 점수 계산, 의류 고정 매칭(Locking), 미충족 조건 처방 등 비즈니스 로직 담당.
- **특징:** 외부 API 의존성(OpenAI, 외부 기상청 API 등)이 100% 배제되어 로컬 오프라인 환경에서 초고속으로 연산됩니다.

### 2) `core/analyzer.py` (시각 및 메타데이터 분석 엔진)
- **역할:** 옷장/구매후보 폴더에 이미지가 새로 추가되었을 때 메타데이터를 스캔하고, Pillow 라이브러리를 활용해 색상을 검출하거나 상/하 이분할 연산을 통해 카테고리를 추론.
- **특징:** 이미지 파일명 규칙을 우선하고, 정보가 없는 경우 이미지 자체 분석 폴백을 사용해 사용자 태깅 피로도를 최소화합니다.

### 3) `config.py` (글로벌 설정)
- **역할:** 각 데이터베이스(JSON) 경로 및 데이터셋 폴더 경로 통합 관리.
- **특징:** 경로 변경이 필요할 때 이 파일 하나만 수정하면 전체 시스템에 일괄 적용됩니다.

## 3. 기능 추가 및 삭제 가이드 (Scalability Guide)

다른 개발자가 기능을 쉽게 확장하거나 특정 기능을 삭제할 수 있도록 확장 포인트(Extension Points)를 설계해 두었습니다.

### ① 새로운 스타일/TPO 조건 추가하기
새로운 추천 조건을 UI와 추천 알고리즘에 반영하고 싶다면 다음 3단계만 진행하면 됩니다.
1. **UI 추가 (`app.py`):** 상세 조건 Expander 영역에 새로운 `selectbox` 또는 `multiselect` 추가.
2. **함수 파라미터 추가 (`core/recommender.py` -> `recommend_outfit`):** 새로운 인자를 파라미터에 추가 (기본값 설정 권장).
3. **스코어링 룰 추가 (`core/recommender.py`):**
   ```python
   # 예시: 원하는 핏(핏감) 가중치 계산 추가
   if desired_fit != "선택 안 함" and desired_fit in item.get("fit", ""):
       item_score += 10
   ```

### ② 특정 조건 기능 제거하기 (예: 활동량 수준 제거)
불필요한 조건을 삭제하고자 할 때는 다음 과정을 거칩니다.
1. **UI 제거 (`app.py`):** `rec_activity` 입력 위젯을 제거합니다.
2. **파라미터 제거 (`core/recommender.py`):** `recommend_outfit`의 `activity_level` 매개변수를 제거하거나 사용하지 않도록 방치합니다.
3. **스코어링 로직 주석 처리/삭제 (`core/recommender.py`):**
   ```python
   # 아래의 활동량 기반 감점 로직을 제거
   # if activity_level == "높음" and item_comfort == "낮음": item_score -= 15
   ```

### ③ 추천 룰 세부 점수 밸런싱 수정
의류 매칭이나 스타일 레퍼런스 선정 시의 점수 가중치를 미세 조정하려면 `core/recommender.py` 내의 점수 가산 상수들만 변경해주면 됩니다.
- **기온 및 계절 적합 가점:** 현 최대 `+30점` 가중치 (freezing_winter/hot_summer에 극단적 외투/반팔 조합 보장)
- **기온 부적합 감점 페널티:** 현 최대 `-30점` 감점 (여름 패딩/겨울 반팔 배제)
- **레퍼런스 스타일 타겟팅 점수:** 현 `+15점` (수정 대상: `core/recommender.py` 내 점수 계산부)
- **선호 색상 일치 점수:** 현 `+12점`
- **의류 계절 부합 점수:** 현 `+10점`
- **의류 스타일(TPO) 부합 점수:** 현 `+5점`

## 4. 최근 긴급 버그 수정 및 개선 사항 (Bug Fix Log)

* **지능형 6단계 기온 연동 및 온도 파싱 패치 (2026-05-20 추가):**
  - 단순 3단계 날씨 분류를 폐지하고, "영하 5도", "기온 30도" 같은 사용자 날씨 텍스트 입력값에서 숫자를 파싱해 6가지 정밀 기온 그룹(`freezing_winter` 등)으로 분류하는 지능형 날씨 해석기 도입.
  - 겨울철 니트 및 패딩 가중치 강화(+30점)와 얇은 가디건 감점 처리, 여름철 긴팔/아우터 감점 페널티(-30점)를 세분화하여 계절 조화의 현실성 극대화.
* **상세 추천 조건 DB 호환 하이브리드 키워드 매핑 (2026-05-20 추가):**
  - "포멀/클래식"과 같은 사용자 입력 위젯 필터가 실제 DB의 라벨 분포(`놈코어(normcore)`, `캐주얼`, `스포티`, `로맨틱`, `레트로`)와 매칭되지 않아 가산점이 누락되던 구조 개선.
  - 슬래시(/) 분할 및 토큰 단위 다중 매칭과 보조 라벨 동적 확장 룰을 탑재한 하이브리드 파서를 구축하여 미충족 판단(`unmet_conditions`) 및 추천 스코어링의 정확도 100% 확보.

* **복합 슬래시(/) 분위기 매칭 오진단 수정:**
  - 5대 상세 조건 중 원하는 분위기를 `"포멀/클래식"` 혹은 `"로맨틱/페미닌"`과 같이 복합 슬래시(/) 문자열로 선택했을 때, 매칭된 레퍼런스 스타일의 mood 필드에 `"포멀"`이나 `"클래식"` 중 하나만 포함되어 있더라도 AI가 이를 인식하지 못하고 `"원하는 분위기 미충족(unmet)"`으로 판정하여 부족하지 않은 옷 구매를 처방하는 오작동이 있었습니다.
  - `core/recommender.py`의 분위기 미충족 진단 로직에서, 선택한 분위기를 슬래시(/) 기준으로 분할하여 각각의 개별 키워드 중 단 하나라도 매칭된 스타일에 존재하면 **정상 매칭(Met)**된 것으로 판정하도록 알고리즘을 보정하고 `test_unmet_conditions_slash_handling` 단위 테스트를 추가해 검증을 완료했습니다.

* **UI 간소화 및 중복 입력 조건 제거:**
  - 메인 패널에 위치하여 화면을 복잡하게 하던 상세 코디 추천 필터들을 좌측 사이드바 `st.expander("⚙️ 상세 조건 설정")` 내부로 이동시켰습니다.
  - 기획 단계의 "어디 가시나요? (TPO)" 텍스트 입력값과 의미·기능 면에서 완전히 중복되던 `"오늘의 코디 분위기 (TPO 무드)"` 선택 위젯을 UI에서 제거하여 사용자 경험을 직관적으로 개선했습니다.

* **저장된 코디 히스토리 로딩 시 NameError 오류 수정:**
  - 이전 저장 코디를 로드할 때 의상 정렬을 위한 `category_order` 변수가 로컬 스코프 외부로 인식되어 `NameError: cannot access free variable 'category_order'`가 발생하는 예외를 발견했습니다.
  - 해당 변수 선언을 `main()` 함수의 최상단 스코프로 인출(Lift)하여 히스토리가 문제없이 복원되도록 해결하였습니다.

* **구매 후보 의류 남성복 전면 교체 (성별 정합성 확보):**
  - 임의 생성된 구매 후보 의류(브라운 코트, 베이지 블레이저)가 여성 의류 사진과 정보로 구성되어 있어, 남성 옷장과 믹스 추천을 돌렸을 때 남녀 의상이 혼합되어 추천되는 문제를 해결했습니다.
  - AI 이미지 생성을 통해 고품질의 **남성용 울 코트 및 베이지 블레이저**로 이미지를 대체하고, `shopping_candidates.json` DB의 성별 정보를 `"Man"`으로 수정하여 코디 매칭의 현실성을 극대화했습니다.

## 5. 결론 및 권장 다음 단계 (Recommended Next Steps)

현재 AI 코디네이션 엔진은 **로컬 데이터 분류 -> 사용자 세부 필터링 -> 스코어링 매칭 -> 안전망(폴백) -> 미충족 진단 및 쇼핑 처방 -> 히스토리 영속화**로 이어지는 견고한 핵심 파이프라인을 온전히 갖추고 있습니다.

**앞으로 다른 개발자가 이 엔진을 넘겨받아 진행하기에 가장 적절한 다음 단계는 다음과 같습니다:**
1. **DB 마이그레이션:** 현재 로컬 JSON 형태의 DB(`my_closet.json` 등)를 클라우드 데이터베이스(RDB, Firebase 등)로 전환하여 다중 사용자 환경 구축.
2. **실제 쇼핑 API 연동:** `complementary_suggestions`에서 부족하다고 진단된 추천 아이템에 대해 가짜 쇼핑 리스트 대신 네이버 쇼핑, 쿠팡 등의 실제 상품 검색 API를 연동하여 실시간 최저가 링크 제공.
3. **Pillow 이미지 프로세싱의 비동기화:** 옷 이미지를 대량으로 스캔할 경우 Streamlit UI가 멈추는 현상을 방지하기 위해 Celery 등의 비동기 작업 큐 도입.
